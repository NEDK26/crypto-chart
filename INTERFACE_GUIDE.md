# 量化图表界面说明

## 1. 界面定位

该页面是一个实时的 BTC/USDT 可视化看盘界面，核心目标是：

- 同一时刻只展示一个 K 线周期
- 将关键技术信息集中在一屏内
- 让价格结构（支撑/阻力）和实时盘口同时可见

当前集成内容包括：

- 实时 K 线图
- 成交量副图（面积受限）
- 支撑位 / 阻力位叠加线
- 布林带快照
- 盘口深度
- 信号流

## 2. 页面分区

### 2.1 顶部栏

- 左上角交易对下拉：可选 `BTC / USDC`、`ETH / USDC`
- 交易对标题：显示当前选中的交易对
- 最新价和涨跌幅显示
- 周期切换按钮：`1分`、`5分`、`1小时`、`4小时`、`1天`

### 2.2 左侧图表区

- 主图：K 线（单周期）
- 主图叠加：`EMA`、`BOLL`
- 副图：`VOL`（成交量）
- 图表下方悬浮信息条：鼠标悬浮 K 线时临时显示时间、OHLC、V、涨跌幅
- 结构线：
  - 支撑位：白色细实线
  - 阻力位：黄色细实线
- 网格：仅保留低透明度水平分割线（无竖向网格、无虚线）

### 2.3 右侧洞察区

- 布林带快照（上轨 / 中轨 / 下轨）
- 关键价位列表（支撑与阻力）
- 深度盘口（价格 / 数量 / 名义金额）

### 2.4 底部信号流

- 按时间倒序显示最新触发事件
- 当前包含：
  - 突破布林上轨 / 下轨
  - 接近支撑位 / 阻力位

## 3. 周期切换机制

当用户点击周期按钮时，会触发完整刷新流程：

1. 重建图表实例
2. 重新拉取该周期历史 K 线
3. 重连该周期的实时 K 线 WebSocket
4. 重新计算支撑/阻力与信号状态

说明：页面始终只渲染一个时间周期，不会多周期并行显示。

## 4. 数据来源

- 历史 K 线（按交易对切换）：
  - `https://api.binance.com/api/v3/klines`
- 实时 K 线（按交易对与周期切换）：
  - `wss://stream.binance.com:9443/ws/{symbol}@kline_{interval}`
- 实时盘口深度（按交易对切换）：
  - `wss://stream.binance.com:9443/ws/{symbol}@depth20@100ms`

## 5. 当前视觉规范

- 支撑位：白色、细线、实线
- 阻力位：黄色、细线、实线
- 背景网格：仅水平线，透明度较低
- 网格线仅作阅读辅助，避免干扰 K 线主体

## 6. 使用建议

- 默认先看 `1分` / `5分` 观察短线节奏
- 再切换 `1小时` / `4小时` 确认结构方向
- `1天` 适合看中期关键位置
- 通过光标悬浮读取单根 K 线的 OHLCV 与涨跌幅细节
- 结合右侧盘口深度和底部信号流提高判断效率
