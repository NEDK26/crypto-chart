# 量化图表界面说明

## 1. 界面定位

该页面是一个实时的 BTC/USDT 可视化看盘界面，核心目标是：

- 同一时刻只展示一个 K 线周期
- 将关键技术信息集中在一屏内
- 让价格结构（支撑/阻力）和实时盘口同时可见

当前集成内容包括：

- 实时 K 线图
- 成交量副图（面积受限）
- 支撑位 / 阻力位叠加线
- 布林带快照
- 盘口深度
- 信号流

## 2. 页面分区

### 2.1 顶部栏

- 左上角交易对下拉：可选 `BTC / USDC`、`ETH / USDC`
- 交易对标题：显示当前选中的交易对
- 最新价和涨跌幅显示
- 周期切换按钮：`1分`、`5分`、`1小时`、`4小时`、`1天`

### 2.2 左侧图表区

- 主图：K 线（单周期）
- 主图叠加：`EMA`、`BOLL`
- 副图：`VOL`（成交量）
- 图表下方悬浮信息条：鼠标悬浮 K 线时临时显示时间、OHLC、V、涨跌幅
- 结构线：
  - 支撑位：白色细实线
  - 阻力位：黄色细实线
- 网格：仅保留低透明度水平分割线（无竖向网格、无虚线）

### 2.3 右侧洞察区

- 布林带快照（上轨 / 中轨 / 下轨）
- 关键价位列表（支撑与阻力）
- 深度盘口（价格 / 数量 / 名义金额）

### 2.4 底部信号流

- 按时间倒序显示最新触发事件
- 当前包含：
  - 突破布林上轨 / 下轨
  - 接近支撑位 / 阻力位

## 3. 周期切换机制

当用户点击周期按钮时，会触发完整刷新流程：

1. 重建图表实例
2. 重新拉取该周期历史 K 线
3. 重连该周期的实时 K 线 WebSocket
4. 重新计算支撑/阻力与信号状态

说明：页面始终只渲染一个时间周期，不会多周期并行显示。

## 4. 数据来源

- 前端优先通过本地行情服务获取数据：
  - 快照 API：`/api/market/snapshot?symbol={symbol}&interval={interval}`
  - 实时 WS：`/ws/market`
- 行情服务再统一连接 Binance：
  - 历史 K 线：`https://api.binance.com/api/v3/klines`
  - 实时 K 线：`wss://stream.binance.com:9443/ws/{symbol}@kline_{interval}`
  - 实时盘口深度：`wss://stream.binance.com:9443/ws/{symbol}@depth20@100ms`

本地联调可一键启动：

```bash
./start-dev.sh
```

也可以手动分别启动两个进程：

```bash
# 终端1：启动行情服务
npm --prefix bot run dev:market

# 终端2：启动前端
npm run dev
```

## 5. 当前视觉规范

- 支撑位：绿色区间带 + 中轴线（按强弱分层）
- 阻力位：粉红区间带 + 中轴线（按强弱分层）
- 可叠加高一级周期关键位：当前周期实线，高一级周期虚线
- 背景网格：仅水平线，透明度较低

## 6. 使用建议

- 默认先看 `1分` / `5分` 观察短线节奏
- 再切换 `1小时` / `4小时` 确认结构方向
- `1天` 适合看中期关键位置
- 通过光标悬浮读取单根 K 线的 OHLCV 与涨跌幅细节
- 结合右侧盘口深度和底部信号流提高判断效率

## 7. 支撑位 / 阻力位说明（补充）

### 7.1 概念说明

- 支撑位：价格下跌到该区域时，更容易出现买盘承接，价格可能减速、止跌或反弹。
- 阻力位：价格上涨到该区域时，更容易出现卖盘压制，价格可能放缓、受阻或回落。
- 这两个价位本质上都是“历史成交密集+反复反应”的价格区域，不是绝对点位。

### 7.2 页面中的识别方式

当前页面中的支撑/阻力来自历史 K 线结构分析，核心流程是：

1. 先在历史 K 线中识别局部高低点（pivot）。
2. 再把价格相近的候选点聚类为同一条结构线（容差按周期自适应，短周期更严格、长周期更宽松）。
3. 按“触发次数 + 最近触达（按 K 线根数衰减）”计算强度分数。
4. 分别保留最有代表性的支撑位和阻力位（每类最多 4 条）。

页面表现：

- 主图叠加：支撑/阻力区间带 + 中轴 + 右侧标签（S/R、距离、触发次数）。
- 右侧“关键价位”面板：展示 S1/R1、强弱、距离百分比与触发次数。
- 底部信号流：当价格接近关键位时会触发接近支撑/阻力提示。

### 7.3 使用建议

- 支撑位附近重点看是否出现放量止跌、盘口买量增强。
- 阻力位附近重点看是否出现放量突破，或冲高回落。
- 建议结合布林带、成交量和盘口深度一起判断，避免单一指标误判。
